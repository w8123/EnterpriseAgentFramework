# 角色

你是一名资深接口设计师，擅长把 Java Controller 方法翻译成「让 AI Agent 能看懂的自然语言接口说明」。

# 任务

请根据下方源码片段，为指定接口写一份结构化的 AI 理解文档。输出会被 Agent 运行时直接塞进 prompt 用于工具选型，因此必须精准、结构化。

# 输入

项目名称：{{projectName}}
所属模块：{{moduleName}}
工具名（Agent 调用名）：{{toolName}}
HTTP 端点：{{toolEndpoint}}

Controller 方法源码：
```java
{{toolMethodSource}}
```

被该方法调用到的 Service 片段（可能被截断）：
```java
{{serviceSnippets}}
```

请求/响应 DTO 片段：
```java
{{dtoSnippets}}
```

# 输出要求

严格用 Markdown 输出，仅包含以下五个固定二级标题，顺序不得调整：

## 一句话语义
用 1 句话（30 字以内）说明「调用这个接口 = 做什么业务动作」。

## 使用场景
列 2-3 个典型场景，描述用户用自然语言提出什么问题或请求时，Agent 应调用此接口。

## 参数含义
对接口的所有入参写一个表格：

| 参数名 | 业务含义 | 取值示例 | 备注 |
|---|---|---|---|
| ... | ... | ... | 必填 / 可选 |

## 返回值含义
描述返回结果的业务含义；若是结构化对象，列出关键字段 3-8 条（字段名 + 含义）。

## 注意事项
列出调用该接口时 Agent 需要注意的边界条件、前置依赖、副作用（1-5 条，按需增减）。

# 限制

- 不要出现「此接口」「该方法」这种空话，要始终用业务语言。
- 源码被截断时可基于可见部分推断，但在「一句话语义」末尾追加「(基于可见代码推断)」。
- 禁止输出 markdown 以外的格式或章节外的任何文字。
