# 任务

请根据下方提供的扫描项目元信息，为该项目写一份「AI 能理解的项目说明文档」，帮助下游 AI Agent 理解这个系统在做什么。

# 输出要求

严格用 Markdown 输出， 5000字以内，包含且仅包含以下四个二级标题，顺序不得调整：

## 业务域
用 2-3 句话概括这个系统所处的业务领域（如"订单中心"、"客户管理"等），并点出核心价值。

## 核心概念
列出 5-8 个本系统里最核心的业务实体/概念（如"订单"、"客户"、"工单"），用短列表，每项一句话解释。

## 典型使用场景
举 2-3 个最有代表性的业务场景（如"一个客户从下单到发货的完整流程"），描述 Agent 在该场景下大概会调用哪些模块。

# 限制

- 不要编造未出现的模块或概念。
- 若某一部分信息不足，可以用「信息不足，需后续补充」明确说明，但不要跳过章节。
- 禁止输出任何与输出要求无关的文字（前言、总结、标题外的一级标题等）。
